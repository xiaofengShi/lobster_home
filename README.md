## 它是什么

**LobsterHive** 是一个基于蜂群智能（Swarm Intelligence）的多 Agent 智能家居系统。不是一个大模型做所有事，而是 **5 只专业蜜蜂 + 1 只蜂后**，各司其职守护你的家。

> 蜜蜂用摇摆舞传递食物方向和距离 → 我们用 JSON 事件传递结构化数据。

## 🐝 蜂群矩阵

| 蜜蜂 | 角色 | 职责 |
|------|------|------|
| 👑🦞 **蜂后 Alan** | Queen | 总调度——不干活，只指挥 |
| 🔭 **侦查蜂** | Scout | 摄像头截图 + VLM 视觉分析 + 传感器 + 天气 |
| ⚔️ **守卫蜂** | Guard | 纯规则引擎安全检测（摔倒/火灾/入侵） |
| 💊 **哺育蜂** | Nurse | 个性化天气关怀、课前提醒、到家通知 |
| 💃 **舞蹈蜂** | Dancer | 统一通知出口（飞书 + 音箱），去重 + 静音 |
| 📖 **筑巢蜂** | Builder | 指纹学习、行为统计、知识持久化 |

**日成本 ≈ ¥4.35** — 只有"看"需要大模型，其他全是规则引擎或轻量推理。

## ⚡ 核心亮点

### 🎯 摇摆舞 = 事件总线

蜜蜂之间通过结构化 JSON 事件通信，完全解耦：

```
门锁触发 → 侦查蜂发布 door_unlock 事件
         → 守卫蜂：已知指纹 + 合理时段 → ✅ 安全
         → 哺育蜂：17:17 = 接小宝回来 → 关怀消息
         → 舞蹈蜂：飞书 "小宝到家啦！🎒" + 音箱播报
         → 筑巢蜂：记录到家时间，学习行为模式
```

### 🔭 两级视觉策略

帧差检测跳过无变化画面，仅在画面变化或传感器触发时调用 VLM。**节省 60-80% VLM 调用**。

### 🛡️ 双层安全检测

守卫蜂第一层**关键词匹配**（<1ms），第二层 **n-gram 语义相似度**兜底。关键词漏检时语义层兜住。

- ⏱️ **<1ms** 延迟 · 💰 **¥0** 成本 · 🎯 **双层兜底**

### 🧬 贝叶斯指纹学习

VLM 视觉特征 + 门锁指纹 ID 交叉验证。**贝叶斯后验概率 > 90%** 自动确认身份。

### ⚡ 并行编排

蜂后使用 ThreadPoolExecutor 并行调度：异常检测 ∥ 早间播报，日报 ∥ 到家 ∥ 晚间。

### 🏥 蜂巢自愈 + Dead-Letter

蜜蜂连续失败 → 自动重置 + 通知管理员。事件消费失败 → dead-letter 记录，不丢事件。
